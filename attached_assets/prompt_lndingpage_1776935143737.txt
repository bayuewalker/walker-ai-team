OPERATIONAL MODEL
====================================================================

The team operates on a truth-centric execution model with two modes.

Normal Mode is the default operating posture. Structured, methodical,
with full review gates. Used when scope requires assessment, risk is
unclear, or the lane is new.

Degen Mode is a fast execution posture activated only by explicit founder
command. Designed to eliminate wasted time from review noise, cosmetic
debates, micro-task fragmentation, and explanation loops. Degen mode
batches fixes, prioritizes implementation over discussion, and pushes
lane closure — while still respecting every safety gate, validation
requirement, and repo truth boundary.

The workflow follows a clear chain:
Mr. Walker sets direction
COMMANDER reads repo truth and forms execution lanes
NEXUS executes through the right specialist
COMMANDER reviews, validates, and merges
Post-merge, all state files are synchronized and the next lane is determined
Every decision is traceable. Every output is evidence-backed.

DESIGN SPECIFICATIONS
====================================================================

Color palette:
- Primary background: deep black or near-black charcoal (#0a0a0a)
- Secondary background: dark gray (#141414 or #1a1a1a)
- Accent color: electric blue (#00a8ff) or emerald green (#00d4aa)
- Text primary: white (#ffffff)
- Text secondary: light gray (#a0a0a0)
- Avoid warm colors entirely

Typography:
- Use Google Fonts: Inter for body, Space Grotesk for headlines
- Headlines bold and large
- Body text clean and readable at 16px base
- Monospace (JetBrains Mono or Fira Code) for technical references
- Load fonts via Google Fonts CDN link in the HTML head

Layout:
- Full-width sections with generous padding (80px to 120px vertical)
- Max content width 1200px centered
- No clutter — every element has breathing room
- Grid-based alignment
- Mobile-first responsive design with breakpoints at 768px and 480px

Animations:
- Subtle scroll-triggered animations using CSS only (no JavaScript libraries)
- Fade-in on sections using IntersectionObserver
- Slight parallax effect on hero background
- No flashy transitions or bouncing elements
- Everything should feel smooth and intentional

Imagery:
- CSS-generated abstract backgrounds (gradients, grid patterns, subtle noise)
- No external image files required
- Geometric shapes suggesting precision and structure
- Animated dot grid or flowing lines in hero section using CSS or minimal JS

Overall feel:
- Like opening a Bloomberg terminal for the first time
- Serious, powerful, clearly built by people who know what they are doing
- Premium without being pretentious
- Technical without being inaccessible
- Confident without being arrogant
