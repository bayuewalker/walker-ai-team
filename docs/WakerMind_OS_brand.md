## 5. Symbol System

| Symbol | Meaning | Used by |
|---|---|---|
| 🔹 Blue diamond | Authority / Director tier | WARP🔹CMD |
| 🔸 Orange diamond | Execution / Team tier | WARP🔸CORE |
| • Bullet | Core team member | WARP•FORGE, WARP•SENTINEL, WARP•ECHO |

**Design rationale:**
- 🔹 and 🔸 = tier indicators (who commands, who leads the team)
- `•` = membership marker (part of CORE, no hierarchy noise)
- System reads like a deliberate versioning/protocol language

> **Note:** Emoji must render consistently across GitHub, Telegram, and dashboard environments.

## 6. Naming Conventions

| Context | Format | Example |
|---|---|---|
| Formal reference | Full name with symbol | WARP🔹CMD, WARP•FORGE |
| Branch prefix | Uppercase, no emoji | `WARP/{feature}` |
| Code / internal | Lowercase, no emoji | `warp-cmd`, `warp-forge` |
| Documentation | Full name with role | `WARP•SENTINEL (Review Agent)` |
| Repo reference | SCREAMING_SNAKE | `WARP_CMD`, `WARP_FORGE` |

### Branch Naming Format

```
WARP/{feature}
```

- Prefix always `WARP/` — uppercase, no exceptions
- `{feature}` is a short hyphen-separated slug
- No dots, no underscores, no date suffix

Correct:
- `WARP/wallet-state-read-boundary`
- `WARP/risk-drawdown-circuit`
- `WARP/briefer-investor-report`

Wrong:
- `NWAP/wallet-state` — deprecated prefix
- `WARP/implement_wallet_state` — underscores
- `WARP/fix-2026-04-28` — date suffix
